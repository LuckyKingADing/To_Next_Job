# pvtlc定位方案精度评估分析任务

## 背景
你正在分析pvtlc（紧耦合定位方案）的精度评估数据。

**重要定义**：
- **正常场景** = 除隧道之外的所有场景，包括开阔场景、半遮挡、双边遮挡、转发器等
- **隧道场景** = GNSS完全不可用的极端场景

重点关注"正常"场景下的精度表现，这是实际日常使用中最常见的场景组合。

## 数据文件说明

### 精度统计文件路径
`.../All_Results/`

### 文件类型
1. **聚合统计文件**：
   - `1_position_precision_aggregated.txt` - 所有子数据集加权平均聚合统计
   - `1_velocity_precision_aggregated.txt` - 速度精度聚合统计

2. **子数据集精度文件**：
   - `pvt_nocamera_{timestamp}_position_precision.txt` - 各子数据集位置精度
   - `pvt_nocamera_{timestamp}_velocity_precision.txt` - 各子数据集速度精度

3. **可视化汇总**：
   - `vis/aggregated/aggregated_summary_table.txt` - 聚合数据汇总表
   - `vis/`目录下的PNG柱状图文件

### 场景分类
**正常场景（重点关注）**：
- 开阔场景：卫星信号良好
- 半遮挡：部分卫星信号受限
- 双边遮挡：卫星信号严重受限
- 转发器：特殊信号环境

**极端场景（对比参考）**：
- 隧道：GNSS完全不可用，纯IMU漂移

### 精度指标含义
| 指标 | 含义 | 单位 |
|------|------|------|
| H-rms | 水平误差均方根 | m |
| H-CEP95 | 95%水平误差概率 | m |
| H-CEP99 | 99%水平误差概率 | m |
| H-max | 最大水平误差 | m |
| L-rms | 横向误差均方根 | m |
| F-rms | 前进方向误差均方根 | m |
| V-rms | 高程误差均方根 | m |

**速度精度**：RMS、CEP50、CEP95、CEP99、Max，单位为m/s

## 分析任务

### 第一部分：正常场景精度基线（核心重点）

从聚合文件提取pvtlc在"正常"场景的精度数据：

1. **提取关键指标**：
   - 正常场景总里程（开阔+半遮挡+双边遮挡+转发器的里程之和）
   - H-rms、H-CEP95、H-CEP99、H-max
   - L-rms、L-CEP95、L-max
   - F-rms、F-CEP95、F-max
   - V-rms、V-CEP95、V-max

2. **精度等级评估标准**：
   - H-rms < 10m → 优秀
   - H-rms 10-15m → 良好
   - H-rms 15-20m → 合格
   - H-rms > 20m → 待改进
   
   - H-CEP95 < 10m → 优秀
   - H-CEP95 10-20m → 良好
   - H-CEP95 > 20m → 待改进

3. **分方向精度对比**：
   - 横向 vs 前进方向精度对比
   - 计算L-rms/F-rms比值

4. **高程精度分析**：
   - V-rms是否正常
   - 分析正常场景下高程误差是否可控

### 第二部分：正常场景各子类型详细分析

分析正常场景下的各个子类型表现：

1. **开阔场景（最佳基准）**：
   - 提取精度值，作为正常场景的最佳性能基准
   - 评估在最优条件下的精度上限

2. **半遮挡场景**：
   - 提取精度值
   - 计算相对于开阔场景的精度衰减比例
   - 评估GNSS部分受限时的适应能力

3. **双边遮挡场景**：
   - 提取精度值

   - 重点分析H-max是否异常

4. **转发器场景**：
   - 提取精度值

5. **正常场景各子类型汇总对比表**

3. **异常识别**：
