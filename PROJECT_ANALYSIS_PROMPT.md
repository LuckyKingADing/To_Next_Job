# 开源项目深度分析提示词模板

> 使用说明：将下方提示词复制，替换 `{项目信息}` 部分为实际项目信息，发送给AI即可

---

## 提示词正文

```
我需要对以下GitHub开源项目进行深度分析，请帮我完成：

## 项目基本信息
- 项目名称：{项目名称}
- GitHub地址：{GitHub URL}
- 项目定位：{自动驾驶/SLAM/定位/感知 等具体方向}
- 主要语言：{C++/Python/等}
- 依赖框架：{ROS/PyTorch/g2o/Ceres 等核心依赖}

## 分析要求

### 一、代码架构与流程分析
请深入分析项目的代码结构，包括但不限于：

1. **项目整体架构**
   - 核心模块划分及职责
   - 模块间的依赖关系和数据流向
   - 配置文件结构和参数管理方式

2. **核心算法流程**
   - 主入口函数及启动流程
   - 关键算法的执行流程（请结合代码文件路径和行号说明）
   - 多线程/异步处理机制（如有）

3. **数据结构设计**
   - 核心数据结构定义
   - 消息通信机制（ROS topic/service/action等）
   - 数据存储格式

### 二、技术亮点与创新点
请重点分析项目的技术特色：

1. **算法创新**
   - 相比传统方法的改进点
   - 是否有论文支撑，简述核心思想
   - 工程化落地的优化技巧

2. **工程实践**
   - 代码规范和设计模式
   - 性能优化手段（并行、SIMD、GPU加速等）
   - 鲁棒性设计（异常处理、边界条件）

3. **可借鉴之处**
   - 代码组织方式
   - 接口设计
   - 测试方法

### 三、面试问题整理（社招方向）

请针对该项目及相关技术领域，整理面试可能问到的深度问题，按难度分级：

#### 基础概念类（校招/初级）
1. {问题} - 参考答案要点：{...}
2. {问题} - 参考答案要点：{...}

#### 工程实践类（社招/中级）
1. {问题} - 参考答案要点：{...}
   - 这里的设计权衡是什么？
   - 实际项目中遇到过什么坑？

#### 架构设计类（高级/架构师）
1. {问题} - 参考答案要点：{...}
   - 如果要扩展到更大规模，你会怎么设计？
   - 有哪些替代方案？各自的优劣？

#### 手撕代码类
1. {问题} - 请给出代码实现思路

### 四、扩展知识图谱
请补充该项目涉及的核心知识点及其关联：

1. **前置知识**（学习该项目前需要掌握的）
2. **关联项目**（同类或互补的开源项目）
3. **延伸方向**（基于该项目可深入学习的方向）

---

## 输出格式要求
请以Markdown格式输出分析报告，包含：
- 代码片段请标注文件路径和行号
- 流程图如需要，请用Mermaid语法绘制
- 复杂概念请配合图示说明
- 面试问题请给出参考答案要点

开始分析吧！
```

## 使用示例

假设你要分析 ORB-SLAM3 项目，填充后如下：

```
我需要对以下GitHub开源项目进行深度分析，请帮我完成：

## 项目基本信息
- 项目名称：ORB-SLAM3
- GitHub地址：https://github.com/UZ-SLAMLab/ORB_SLAM3
- 项目定位：视觉SLAM（支持单目、双目、RGB-D，支持视觉+IMU融合）
- 主要语言：C++
- 依赖框架：OpenCV、Eigen、g2o、DBoW2
...
```

## 可选补充提示词

如果需要针对性分析某个方面，可以追加以下提示：

### 追加 - 性能分析方向
```
请额外分析：
1. 系统的性能瓶颈在哪里？
2. 各模块的时间复杂度分析
3. 内存使用特点
4. 实时性保障措施
```

### 追加 - 对比分析方向
```
请额外对比该项目与 {对比项目名}：
1. 架构设计差异
2. 算法选择差异
3. 性能对比（精度、速度）
4. 适用场景差异
```

### 追加 - 改进建议方向
```
请额外给出：
1. 代码可改进之处
2. 可优化空间
3. 如果是你来重构，会怎么设计？
```

### 追加 - 论文复现方向
```
请结合论文 {论文名} 分析：
1. 论文中的算法与代码实现的对应关系
2. 代码中是否完整复现了论文？
3. 有哪些工程化调整？
```

## 面试问题分类参考

| 类别 | 适合岗位 | 问题深度 |
|------|---------|---------|
| 基础概念类 | 校招、初级工程师 | 考察基础知识掌握 |
| 工程实践类 | 社招、中级工程师 | 考察实际项目经验 |
| 架构设计类 | 高级工程师、架构师 | 考察系统设计能力 |
| 手撕代码类 | 所有岗位 | 考察编码能力 |
