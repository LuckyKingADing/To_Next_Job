## 一、方向定位

### 核心定位
你接下来要找的方向，核心是 **"多传感器紧组合融合定位"**，在自动驾驶量产车和 Robotaxi 场景中需求量大、工业落地成熟。与纯视觉方案（L4 Robotaxi）相比，更偏向 **量产路线**（如华为车 BU、小米汽车、理想、比亚迪等）。

### 与 SLAM 方向的区别

| 维度 | GNSS/IMU/ODO 融合定位 | 视觉 SLAM / 激光 SLAM |
|---|---|---|
| 核心场景 | 室外开阔道路 | 室内/室外复杂场景 |
| 主传感器 | GNSS（室外有绝对精度）| Camera / LiDAR（无绝对参考）|
| 绝对精度 | 有（RTK 可达厘米级）| 无（累积误差）|
| 室内可用性 | 差（GNSS 失效）| 好 |
| 算法难点 | 紧组合建模、模糊度固定 | 特征匹配、闭环检测 |
| 工业落地 | 成熟（Apollo 标配）| 逐步落地 |

### 技能 2：卡尔曼滤波（EKF / ESKF）

**为什么必须掌握**
GNSS/IMU/ODO 融合定位，本质上就是一个 **状态估计问题**，卡尔曼滤波是最核心的工具。ESKF 是自动驾驶定位领域最常用的滤波方法，面试中出现频率极高。

**需要掌握的深度**

```
必须精通：
- KF 完整推导：从贝叶斯估计到卡尔曼增益
- EKF：非线性线性化（一阶泰勒展开），雅可比矩阵的计算
- ESKF：误差状态建模，误差注入，数值稳定性优势
- Q/R 调参：协方差矩阵的物理意义，Q 大会怎样，R 大会怎样
- 状态量设计：位置/速度/姿态/零偏，如何选择状态量

最好了解：
- UKF：Sigma 点采样，不依赖雅可比，适用于强非线性
- PF（粒子滤波）：了解原理，融合定位中一般不用但偶尔被问到
```

**学习路径**

```
第 1 步（3天）：
  → 阅读《Probabilistic Robotics》Chapter 4
  → 在纸上完整推导一维 KF（5 个公式，每个公式推导 3 遍）
  → 用 Python 实现一维匀速模型 KF

第 2 步（3天）：
  → 推导三维 KF，从一维推广
  → 实现三维匀速模型位置跟踪
  → 参数实验：Q/R 比例对滤波效果的影响

第 3 步（5天）：
  → 阅读《State Estimation for Robotics》ESKF 章节
  → 在纸上完整推导 ESKF（重点：误差状态传播、误差注入）
  → 实现 ESKF，处理 IMU 数据，观察姿态估计

第 4 步（2天）：
  → UKF：理解 Sigma 点采样，UT 变换
  → 对比 EKF vs UKF 在非线性场景的精度差异
```

**检验标准**
- [ ] 能闭眼在纸上完整推导 EKF（10 分钟内）
- [ ] 能解释 ESKF 为什么比 EKF 更适合定位
- [ ] 能手写 ESKF 代码，处理一段 IMU 数据
- [ ] 能解释 Q/R 矩阵调参的物理意义

**推荐资料**
- 📖 《Probabilistic Robotics》Chapter 4（必读，圣经）
- 📖 《State Estimation for Robotics》（必读，公式最详细）

### 技能 3：IMU 预积分与噪声建模

**为什么必须掌握**
IMU 是融合定位的核心传感器，提供高频姿态变化率。在 GNSS 信号短暂中断（隧道、城市峡谷）时，IMU 主导定位。在紧组合中，IMU 预积分量是连接两个关键帧之间的关键约束。

**需要掌握的深度**

```
必须精通：
- IMU 测量模型：加速度 + 角速度的真实物理含义
- 随机噪声建模：白噪声（White Noise）、随机游走（Random Walk）
- Allan Variance：读懂 IMU 参数手册（ARW、Bias Instability、VRW）
- 零偏（Bias）建模：随机游走过程，贝叶斯估计
- IMU 预积分：核心中的核心，推导预积分量（Δv、Δp、Δq）
- 预积分残差：对 bias 的雅可比，滑动窗口中的更新

能理解：
- 离散化方法：Euler 法、RK4 法
- 器件选型：BMI088、ADIS16465 的性能差异
```

**学习路径**

```
第 1 步（3天）：
  → 四元数复习：乘法、逆、共轭、旋转变换
  → IMU 测量方程推导（理想 vs 实际）

第 2 步（4天）：
  → Allan Variance 原理：找一款 IMU 手册，分析参数
  → 随机噪声 vs 随机游走的区别
  → 在纸上推导 Allan Variance 曲线的拟合

第 3 步（5天）：
  → 精读 Foster & Love《On-Manifold Preintegration》论文
  → 在纸上完整推导预积分量：Δv、Δp、Δq
  → 推导预积分残差对 bias 的雅可比矩阵

第 4 步（3天）：
  → 读 VINS-Mono 的 integration_base.h 和 imu_factor.h
  → 对照论文看代码，理解 VINS 中预积分的实现
```

**检验标准**
- [ ] 能用 Allan Variance 解释 IMU 参数手册中的数据
- [ ] 能完整推导 IMU 预积分量
- [ ] 能解释预积分残差在优化中的作用
- [ ] 能说出一款 IMU 的具体参数和选型建议

**推荐资料**
- 📖 Foster & Love《On-Manifold Preintegration》（必读论文）
- 📖 《State Estimation for Robotics》IMU 相关章节
- 📖 《惯性导航原理》（中文，硬件层面理解）
- 💻 VINS-Mono `integration_base.h`（最好的预积分学习代码）

### 技能 4：多传感器融合（松组合 + 紧组合）

**为什么必须掌握**
这是 GNSS/IMU/ODO 融合方向 **最核心的技能**，也是面试最高频考点。你目前做的是松组合，紧组合是接下来的必备能力。

**松组合（你已有基础）**

```
你目前做的方向，面试要能讲出深度：
- GNSS 位置/速度的协方差怎么来的
- 松组合的 EKF 状态量设计
- IMU 机械编排（Mechanical Arrangement）：用 IMU 积分预测状态
- 松组合的优缺点（卫星数少时精度急剧下降）
```

**紧组合（重点突破）**

```
紧组合的难度更高，但面试官最看重这部分：

第一步：GNSS 原始观测量
- 伪距（Code Pseudorange）：精度 ~m 级
- 载波相位（Carrier Phase）：精度 ~cm 级，但有模糊度
- 多普勒（Doppler）：精度高，反应速度变化

第二步：双差（Double Difference）
- 站间单差：消除接收机钟差
- 星间双差：消除卫星钟差
- 双差载波相位：模糊度是整数

第三步：模糊度固定（Ambiguity Resolution）
- 浮点解 → 整数解的搜索过程
- LAMDA 算法（最常用）
- 固定解 vs 浮点解：固定解精度高 ~10 倍

第四步：紧组合 EKF 设计
- 状态量：位置、速度、姿态、IMU 零偏、模糊度（可选）
- 观测量：伪距残差、载波相位残差
- 雅可比矩阵的构建

第五步：结合同事的 Apollo 紧组合代码学习
```

**融合框架（ODO 轮速计）**

```
ODO 在融合中的作用：
- 提供速度信息，辅助 IMU 积分
- 非完整性约束（NHC）：假设车辆在路面上行驶
- 比例因子标定：轮速计的尺度误差
- 与 IMU 融合的方式：速度观测量
```

**学习路径**

```

  → 梳理你现有项目中松组合的每个细节
  → 理解 GNSS 位置/速度协方差的来源
  → 在 Apollo 中找到松组合实现代码

第 2 步（5天）：GNSS 观测量原理
  → 伪距、载波相位、多普勒的测量方程
  → 双差推导，在纸上写清楚
  → 理解模糊度整周特性

第 3 步（5天）：紧组合原理
  → 紧组合 EKF 状态量设计
  → 观测量雅可比矩阵推导
  → LAMDA 算法步骤
  → 对比松/紧组合优缺点
```

### 技能 5：因子图与非线性优化

**为什么必须掌握**
紧组合从滤波器方法迁移到因子图方法，是当前工业界的趋势。Apollo 最新的融合定位就在往因子图方向演进，面试中因子图相关问题出现频率越来越高。

**需要掌握的深度**

```
核心概念：
- 因子图：变量节点 + 因子节点
- 非线性最小二乘：min Σ ||r(x)||²
- 高斯-牛顿 / LM 迭代求解

GNSS/IMU/ODO 场景下：
- IMU 预积分因子
- GNSS 伪距/相位因子
- ODO 速度因子
- 边缘化（Marginalization）：Schur Complement

工程实现：
- GTSAM：上手快，文档好
- g2o：SLAM 圈最常用
- iSAM/iSAM2：Apollo 用的增量平滑
```

**学习路径**

```
第 1 步（4天）：因子图基础
  → 用笔纸画 IMU + GPS 融合的因子图
  → 理解为什么因子图是稀疏的
  → 理解因子图的"统一表示"作用

第 2 步（5天）：非线性优化理论
  → 推导高斯-牛顿法：从目标函数到增量方程
  → 理解雅可比矩阵在优化中的作用
  → LM 算法：信赖域，稳定性
  → 在纸上推导完整的 GN/LM 迭代步骤

第 3 步（5天）：滑动窗口与边缘化
  → 为什么需要滑动窗口
  → 边缘化的数学本质：Schur Complement
  → 边缘化引入的误差如何控制
  → 读 VINS-Mono 中的滑窗实现

第 4 步（7天）：GTSAM 实战
  → 安装 GTSAM，配置环境
  → 用 GTSAM 实现一个 IMU + GPS 融合的简单 demo
  → 对比因子图方法 vs 滤波方法的精度
```

**检验标准**
- [ ] 能画出一个包含 IMU/GPS/ODO 的完整因子图
- [ ] 能解释 Schur Complement 在边缘化中的作用
- [ ] 能用 GTSAM 实现一个简单的多传感器融合 demo
- [ ] 能对比滤波方法和因子图方法的异同和适用场景

**推荐资料**
- 📖 《State Estimation for Robotics》（Barfoot）（必读，体系最完整）
- 📖 《Factor Graphs and GTSAM》（入门 GTSAM 必读）
- 💻 GTSAM：文档优秀，示例丰富
- 💻 g2o：SLAM 圈主流工具
- 🎬 港科大 VIO 课程（Bilibili）：因子图理论讲解
