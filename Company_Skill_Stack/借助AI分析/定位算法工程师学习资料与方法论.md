## 一、学习方法论总纲

### 核心原则

**理论 + 代码 + 实践 三位一体**，缺任何一环都会导致面试时底气不足。

```
理论层面：理解原理，能推导公式，能解释trade-off
代码层面：能独立实现核心算法，不依赖开源库
实践层面：跑通开源项目，能分析数据，能做改进
```

### 学习节奏建议

| 阶段 | 时间 | 核心目标 |
|---|---|---|
| 夯实基础 | 5 月 | 卡尔曼滤波 + C++ + IMU 理论 |
| 体系构建 | 6 月 | 紧组合 + 因子图 + GNSS 系统 |
| 综合提升 | 7 月 | VIO + LiDAR SLAM + Apollo 深度 |
| 实战打磨 | 8 月 | 开源项目 + 简历 + 面试 |

### 每项技能的标准学习流程

```
① 理解基本概念（1-2天）
  → 查 Wikipedia / 博客，建立直观认识

② 推导核心公式（3-5天）
  → 手动推导，在纸上写出来
  → 理解每一步的物理/数学意义

③ 实现核心代码（5-7天）
  → 不参考别人的实现，自己写一遍
  → 能跑通简单的 1D/2D demo

④ 对比开源实现（2-3天）
  → 看成熟开源库（如 Eigen / g2o）怎么写的
  → 理解工程化实现的细节

⑤ 总结输出（1-2天）
  → 写成博客 / 笔记
  → 能用自己的话向别人讲清楚
```

### 学习方法

**IMU 测量模型（3-5天）**

```
理想模型：加速度 = 真实加速度 - 重力 - 零偏
实际模型：加上比例因子误差 + 噪声
四元数/旋转矩阵表示旋转（优先四元数，更紧凑）
```

**Allan 方差分析（3-5天）**

```
从器件手册中读懂：Angle Random Walk (ARW)
零偏不稳定性 (Bias Instability)
速度随机游走 (VRW)
用 Allan Variance 曲线拟合这些参数
理解 IMU 选型时这些参数的意义
```

**IMU 预积分（1周）**

```
为什么需要预积分：避免重复积分
关键思想：积分建立在关键帧之间，而不是全局状态
预积分量的定义：Δv, Δp, Δq
预积分残差推导（核心！）：J是对谁求导的？
Jacobian 在滑动窗口中的更新方式
```

### 学习方法

**松组合（已有基础，理论提升）（1周）**

```
你已经会用了，但面试要能讲清楚：
- GNSS 位置/速度的协方差怎么来的
- 松组合中 EKF 的状态量是什么
- IMU 机械编排（Mechanical Arrangement）
- 与纯 IMU 航位推算的对比
```

**紧组合（重点突破）（2周）**

```
第一步：理解 GNSS 原始观测量（伪距、载波相位、多普勒）
第二步：理解双差（Double Difference）消除钟差
第三步：理解模糊度固定（Ambiguity Resolution）
第四步：紧组合的状态量设计
第五步：观测量模型怎么建立（雅可比矩阵）
第六步：结合同事代码，看懂 Apollo 的实现
```

**对比总结（面试必考）**

```
松组合优势：实现简单，计算量小
松组合劣势：卫星数少时精度急剧下降

紧组合优势：卫星数少（>4颗）时仍能工作，精度更高
紧组合劣势：计算量大，模糊度固定复杂

什么时候用：城市峡谷等卫星遮挡场景必须紧组合
开阔地带：松组合足够
```

- [ ] 能解释伪距、载波相位、多普勒三种观测量的特点

### 学习方法

**因子图基础（1周）**

```
变量节点：状态量（位姿、路标）
因子节点：观测量（IMU、GPS、视觉）
边的含义：约束关系
从滤波到因子图：为什么融合问题是稀疏的
用纸笔画几个简单的因子图（3-4个节点）
```

**非线性最小二乘（1周）**

```
目标函数：min ||f(x)||²
高斯-牛顿（GN）：用雅可比近似 Hessian
LM 算法：信赖域，稳定性更好
求解流程：线性化 → 构建增量方程 → 求解 → 更新
```

**滑动窗口与边缘化（1周）**

```
为什么需要滑动窗口：计算量随时间增长
边缘化的物理意义：把旧状态"压缩"成先验
边缘化的实现：Schur Complement
边缘化引入的误差：如何控制
```

**g2o / GTSAM 实战（2周）**

```
g2o: 定义顶点（继承 BaseVertex）、定义边（继承 BaseBinaryEdge）
GTSAM: 用 Factors 和 Values 构建图
用 GTSAM/g2o 复现一个简单的 IMU-GPS 融合例子
对比滤波方法和因子图方法的精度/实时性差异
```

## 十二、面试准备

### 算法题（LeetCode）

定位算法岗位一般 medium 题目居多，重点刷：

```
数据结构：数组、哈希、链表、树、图
算法：二分、双指针、滑动窗口、BFS/DFS、动态规划
高频题型：矩阵搜索、路径规划相关题目
频率：每天 1-2 题，保持手感，面试前 1 个月集中刷
```

### 定位专业知识面试题库

常见面试题分类整理：

**滤波类**
- 手写卡尔曼滤波
- EKF 和 ESKF 的区别
- UKF 和 EKF 的区别和使用场景

**融合类**
- 松组合和紧组合的区别
- 为什么紧组合在卫星数少时更好
- IMU 和 GPS 融合的时机

**理论类**
- IMU 预积分的物理意义
- 模糊度固定的原理
- 因子图比滤波好在哪里

**工程类**
- Apollo 定位模块的数据流
- 定位精度下降的排查思路
- 传感器标定的流程

- 建议 8 月开始，每周一次 Mock 面试
- 可以找同学互相问，或者用录音自己复盘
